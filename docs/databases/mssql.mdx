# Microsoft SQL Server Connections

TablePro supports Microsoft SQL Server 2017 and later via FreeTDS. This covers SQL Server instances running on Windows, Linux, or in Docker, as well as Azure SQL Database.

<Warning>
FreeTDS must be built before connecting to SQL Server. Run `scripts/build-freetds.sh` once to compile the required library.
</Warning>

## Quick Setup

<Steps>
  <Step title="Build FreeTDS">
    Run `scripts/build-freetds.sh` from the repository root if you haven't already
  </Step>
  <Step title="Open Connection Form">
    Click **New Connection** from the Welcome screen or **File** > **New Connection**
  </Step>
  <Step title="Select SQL Server">
    Choose **SQL Server** from the database type selector
  </Step>
  <Step title="Enter Connection Details">
    Fill in host, port, username, password, and database name
  </Step>
  <Step title="Test and Connect">
    Click **Test Connection**, then **Create**
  </Step>
</Steps>

## Connection Settings

### Required Fields

| Field | Description | Default |
|-------|-------------|---------|
| **Name** | Connection identifier | - |
| **Host** | Server hostname or IP address | `localhost` |
| **Port** | SQL Server port | `1433` |
| **Username** | SQL Server login name | `sa` |
| **Password** | Login password | - |
| **Database** | Database name to connect to | - |

<Note>
TablePro connects using SQL Server Authentication (username and password). Windows Authentication is not supported.
</Note>

## Example Configurations

### Local Development Server

```
Name:     Local SQL Server
Host:     localhost
Port:     1433
Username: sa
Password: (your password)
Database: master
```

### Docker SQL Server Container

Start a SQL Server 2022 container for local testing:

```bash
docker run \
  -e "ACCEPT_EULA=Y" \
  -e "SA_PASSWORD=YourStrong@Passw0rd" \
  -p 1433:1433 \
  --name sql-server \
  -d mcr.microsoft.com/mssql/server:2022-latest
```

Then connect with:

```
Name:     Docker SQL Server
Host:     localhost
Port:     1433
Username: sa
Password: YourStrong@Passw0rd
Database: master
```

<Tip>
The `SA_PASSWORD` must meet SQL Server's complexity requirements: at least 8 characters with uppercase, lowercase, digits, and symbols.
</Tip>

### Remote Server

```
Name:     Production SQL Server
Host:     sql.example.com
Port:     1433
Username: app_user
Password: (secure password)
Database: myapp_production
```

### Azure SQL Database

```
Name:     Azure SQL
Host:     myserver.database.windows.net
Port:     1433
Username: admin@myserver
Password: (your password)
Database: mydb
```

<Note>
Azure SQL Database requires the username to include the server suffix (e.g., `user@servername`) or just the username depending on your authentication configuration.
</Note>

## Features

### Schema Selection

SQL Server organizes objects into schemas within a database. The default schema is `dbo`. TablePro displays all schemas your login has access to and lists tables within each schema.

You can switch schemas using the database switcher (**⌘K**), which shows available schemas for the current database.

### Table Browsing

For each table, TablePro shows:

- **Structure**: Columns with SQL Server data types, nullability, and default values
- **Indexes**: Clustered and non-clustered indexes
- **Foreign Keys**: Relationships to other tables
- **DDL**: The table definition including constraints

### Query Editor

Write and execute T-SQL queries in the SQL editor. Pagination uses SQL Server's standard offset syntax:

```sql
-- Paginated results
SELECT *
FROM dbo.orders
ORDER BY order_id
OFFSET 0 ROWS FETCH NEXT 50 ROWS ONLY;

-- Row count estimate (fast, uses statistics)
SELECT SUM(p.rows) AS row_count
FROM sys.partitions p
JOIN sys.tables t ON p.object_id = t.object_id
WHERE p.index_id IN (0, 1)
  AND t.name = 'orders';

-- View definition
SELECT OBJECT_DEFINITION(OBJECT_ID('dbo.my_view'));

-- List all tables in the current database
SELECT TABLE_SCHEMA, TABLE_NAME
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_TYPE = 'BASE TABLE'
ORDER BY TABLE_SCHEMA, TABLE_NAME;
```

### Schema Editing

Modify table structure via T-SQL ALTER TABLE statements in the query editor. SQL Server uses bracket notation for identifiers:

```sql
-- Add a column
ALTER TABLE [dbo].[users] ADD [last_login] DATETIME2 NULL;

-- Rename a column (requires sp_rename)
EXEC sp_rename 'dbo.users.old_name', 'new_name', 'COLUMN';

-- Drop a column
ALTER TABLE [dbo].[users] DROP COLUMN [legacy_field];

-- Add an index
CREATE INDEX [IX_users_email] ON [dbo].[users] ([email]);
```

### Foreign Keys

TablePro displays foreign key relationships in the table structure view. To inspect foreign keys manually:

```sql
SELECT
    fk.name AS fk_name,
    OBJECT_NAME(fk.parent_object_id) AS parent_table,
    COL_NAME(fkc.parent_object_id, fkc.parent_column_id) AS parent_column,
    OBJECT_NAME(fk.referenced_object_id) AS referenced_table,
    COL_NAME(fkc.referenced_object_id, fkc.referenced_column_id) AS referenced_column
FROM sys.foreign_keys fk
JOIN sys.foreign_key_columns fkc ON fk.object_id = fkc.constraint_object_id
ORDER BY parent_table, fk_name;
```

## Authentication

TablePro uses SQL Server Authentication only. Provide a SQL Server login name and password in the connection form.

**Windows Authentication** (Integrated Security / Kerberos) is not supported.

To create a SQL Server login:

```sql
-- Create a login
CREATE LOGIN app_user WITH PASSWORD = 'StrongPassword1!';

-- Create a database user for that login
USE mydb;
CREATE USER app_user FOR LOGIN app_user;

-- Grant permissions
ALTER ROLE db_datareader ADD MEMBER app_user;
ALTER ROLE db_datawriter ADD MEMBER app_user;
```

## Troubleshooting

### Connection Refused

**Symptoms**: "Unable to connect" or timeout when connecting

**Causes and Solutions**:

1. **SQL Server not running or not accepting TCP connections**

   SQL Server Browser and TCP/IP protocol must be enabled:
   - Open SQL Server Configuration Manager
   - Enable TCP/IP under "SQL Server Network Configuration"
   - Ensure SQL Server service is running

2. **Port 1433 blocked by firewall**
   - For Windows Firewall, add an inbound rule for TCP port 1433
   - For cloud VMs, check the security group / firewall rules

3. **Docker container not started**
   ```bash
   docker start sql-server
   docker logs sql-server
   ```

### Login Failed

**Symptoms**: "Login failed for user" error

**Solutions**:

1. Verify the username and password
2. Confirm SQL Server Authentication is enabled (not Windows-only mode):
   ```sql
   -- Check server authentication mode (1 = Windows only, 2 = mixed mode)
   SELECT SERVERPROPERTY('IsIntegratedSecurityOnly');
   ```
3. Enable mixed-mode authentication in SQL Server Management Studio under Server Properties > Security, then restart the SQL Server service

### FreeTDS Not Found

**Symptoms**: "FreeTDS library not found" or linker error on launch

**Solution**: Run the build script from the repository root:

```bash
scripts/build-freetds.sh
```

This compiles FreeTDS and places the static library where the Xcode project expects it.

## Known Limitations

- **Windows Authentication not supported.** Only SQL Server Authentication (username + password) works.
- **Named instances** (e.g., `HOSTNAME\INSTANCENAME`) are not yet supported. Connect via IP address and explicit port instead.
- **`NTEXT`, `TEXT`, `IMAGE` columns** (deprecated since SQL Server 2005) may show limited editing support. Use `NVARCHAR(MAX)`, `VARCHAR(MAX)`, and `VARBINARY(MAX)` in new schemas.
