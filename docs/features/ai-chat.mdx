# AI Chat

TablePro includes a built-in AI chat panel that helps you write, explain, and fix SQL queries. It supports multiple AI providers including Claude (Anthropic), OpenAI, Gemini (Google), OpenRouter, Ollama, and custom endpoints.

## Opening the AI Chat Panel

Open the AI chat panel using any of these methods:

| Method | Action |
|--------|--------|
| Keyboard shortcut | `Cmd+Shift+L` |
| Toolbar | Click the AI button (sparkles icon) |
| Menu | **View** > **Toggle AI Chat** |

The chat panel appears on the right side of the editor, alongside the data inspector.

## Chatting with AI

1. Type your question or request in the input field at the bottom
2. Press **Enter** or click the send button to send
3. The AI responds with context-aware suggestions based on your current database connection

The AI automatically receives context about:
- Your database type (MySQL, PostgreSQL, SQLite)
- Database schema (tables, columns, types, foreign keys)
- Your current query in the editor (when enabled)

<Tip>
The AI generates SQL appropriate for your specific database dialect, using correct identifier quoting and syntax.
</Tip>

### Markdown Responses

AI responses are rendered with full Markdown support including:
- **Formatted text** — headers, bold, italic, lists
- **SQL code blocks** — with syntax highlighting and action buttons
- **Inline code** — for table names, column names, and SQL keywords

## Code Blocks

When the AI responds with SQL code blocks:

- **Copy** — copies the SQL to your clipboard
- **Insert to Editor** — inserts the SQL directly into your active editor tab

## Token Usage

After each AI response completes, the token count is displayed below the assistant message bubble:

- **Input tokens** — tokens sent to the AI (your messages + context)
- **Output tokens** — tokens generated by the AI

This helps you monitor API usage and costs per message.

## Conversation History

AI conversations are automatically saved to disk and restored when you relaunch TablePro. Manage conversations from the chat panel header:

| Button | Action |
|--------|--------|
| **History** (clock icon) | Browse and switch between past conversations |
| **New Chat** (pencil icon) | Start a new conversation |
| **Clear** (trash icon) | Delete the current conversation |

Conversations are stored as JSON files in `~/Library/Application Support/TablePro/ai_chats/`.

<Tip>
Conversations are automatically titled using the first message you send, making them easy to find later.
</Tip>

## Retry and Regenerate

When something goes wrong or you want a different response:

- **Retry** — if AI generation fails (network error, rate limit, etc.), an inline "Retry" button appears below the last user message. Click it to re-send the same request.
- **Regenerate** — after a successful response, a "Regenerate" button appears below the assistant message. Click it to remove the response and stream a new one.

## Streaming

AI responses stream in real-time as they're generated. You can:
- Watch the response build token by token — a three-dot typing indicator shows while waiting for the first token
- Click **Stop** to cancel a streaming response at any time
- Scroll up to read previous messages without being forced back to the bottom — auto-scroll only activates when you're already viewing the latest content
- Start a new message while viewing previous responses

## Supported Providers

TablePro supports six types of AI providers:

| Provider | Endpoint | API Key Required |
|----------|----------|------------------|
| **Claude** (Anthropic) | `https://api.anthropic.com` | Yes |
| **OpenAI** | `https://api.openai.com` | Yes |
| **OpenRouter** | `https://openrouter.ai/api` | Yes |
| **Gemini** (Google) | `https://generativelanguage.googleapis.com` | Yes |
| **Ollama** | `http://localhost:11434` | No |
| **Custom** | Any OpenAI-compatible endpoint | Optional |

<Tip>
**Ollama Auto-Detection**: If Ollama is running locally when TablePro launches, it is automatically detected and registered as an AI provider with the first available model. You don't need to configure it manually.
</Tip>

<Note>
You can configure multiple providers simultaneously and route different AI features to different providers.
</Note>

## Configuring AI Providers

Set up AI providers in **Settings** (`Cmd+,`) > **AI**:

### Adding a Provider

1. Click **Add Provider** in the Providers section
2. Choose a provider type (Claude, OpenAI, Gemini, OpenRouter, Ollama, or Custom)
3. Enter your API key (the endpoint is pre-filled based on the provider type)
4. Specify a model name — or click **Fetch** to retrieve available models from the provider. Models are automatically fetched when you change provider settings.
5. Click **Test** to verify the connection

### API Key Security

API keys are stored securely in your macOS Keychain — they are never stored in plain text or in preferences files.

### Feature Routing

Route specific AI features to different providers:

| Feature | Description |
|---------|-------------|
| **Chat** | General AI chat conversations |
| **Explain Query** | Query explanation requests |
| **Optimize Query** | Query performance optimization suggestions |
| **Fix Error** | Error diagnosis and fix suggestions |
| **Inline Suggestions** | Inline code suggestions |

Each feature can use a specific provider or fall back to the default (first enabled provider).

## Context Settings

Control what database context is sent to the AI:

| Setting | Default | Description |
|---------|---------|-------------|
| **Include database schema** | On | Send table and column definitions |
| **Include current query** | On | Send the query from the active editor tab |
| **Include query results** | Off | Send recent query results |
| **Max schema tables** | 20 | Maximum number of tables to include in context |

## Per-Connection AI Policy

Control AI access on a per-connection basis from the connection form:

| Policy | Description |
|--------|-------------|
| **Use Default** | Follow the global default policy from AI settings |
| **Always Allow** | AI features are always available for this connection |
| **Ask Each Time** | Prompt before sending data to AI |
| **Never** | Disable AI features entirely for this connection |

This is useful for restricting AI on connections to sensitive production databases.

## Editor Integration

AI features are tightly integrated into the SQL editor workflow:

### Context Menu

Right-click on selected SQL in the editor to access AI features:

- **Explain with AI** — sends the selected SQL to AI for a detailed explanation
- **Optimize with AI** — asks AI to suggest performance improvements

### Keyboard Shortcuts

| Action | Shortcut | Description |
|--------|----------|-------------|
| Toggle AI Chat | `Cmd+Shift+L` | Open or close the AI chat panel |
| Explain with AI | `Cmd+L` | Explain selected SQL (or current query) |
| Optimize with AI | `Cmd+Option+L` | Optimize selected SQL (or current query) |

All AI shortcuts are customizable in **Settings** > **Keyboard** under the **AI** category.

<Tip>
When no text is selected, `Cmd+L` and `Cmd+Option+L` use the entire query from the active editor tab.
</Tip>

### Ask AI to Fix Errors

When a query execution fails, the error dialog includes an **Ask AI to Fix** button. Clicking it:

1. Opens the AI chat panel (if not already open)
2. Sends both the failed query and error message to AI
3. AI analyzes the error and suggests a corrected query
