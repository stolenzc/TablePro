# Tham chiếu URL Kết nối

TablePro hỗ trợ các URL kết nối cơ sở dữ liệu chuẩn để nhập kết nối, mở trực tiếp từ trình duyệt hoặc terminal, và cấu hình SSH tunnel. Trang này bao gồm mọi scheme, định dạng và query parameter được hỗ trợ.

## URL Scheme

| Scheme | Cơ sở dữ liệu |
|--------|----------|
| `postgresql://` | PostgreSQL |
| `postgres://` | PostgreSQL (alias) |
| `mysql://` | MySQL |
| `mariadb://` | MariaDB |
| `sqlite://` | SQLite |
| `mongodb://` | MongoDB |
| `mongodb+srv://` | MongoDB (SRV) |
| `redis://` | Redis |
| `rediss://` | Redis với TLS |
| `redshift://` | Amazon Redshift |
| `mssql://` | Microsoft SQL Server |
| `sqlserver://` | Microsoft SQL Server (alias) |

Thêm `+ssh` vào bất kỳ scheme nào (trừ SQLite) để sử dụng SSH tunnel:

```
postgresql+ssh://
mysql+ssh://
mariadb+ssh://
```

## Định dạng Chuẩn

```
scheme://[username[:password]@]host[:port][/database][?param=value&...]
```

**Ví dụ:**

```
postgresql://alice:secret@db.example.com:5432/myapp
mysql://root@localhost/shop
mongodb://user:pass@mongo.host:27017/analytics?authSource=admin
redis://:password@cache.host:6379/1
sqlite:///Users/alice/data/local.db
```

<Tip>
Mật khẩu chứa ký tự đặc biệt (`@`, `#`, `%`, `:`) phải được percent-encode. Ví dụ: `p@ss#word` trở thành `p%40ss%23word`.
</Tip>

## Định dạng SSH Tunnel

URL SSH tunnel mã hóa cả thông tin SSH và cơ sở dữ liệu trong một chuỗi:

```
scheme+ssh://[ssh_user@]ssh_host[:ssh_port]/[db_user[:db_pass]@]db_host[:db_port][/database][?params]
```

**Ví dụ:**

```
postgresql+ssh://deploy@bastion.host:22/dbuser:dbpass@internal-pg:5432/mydb
mysql+ssh://ec2-user@jump.host/root:secret@10.0.0.5/shop
mariadb+ssh://admin@ssh.host/maria_user@db.internal/store?usePrivateKey=true
```

Nếu `db_host` bị bỏ qua, mặc định là `127.0.0.1`.

## Query Parameter

### Danh tính Kết nối

| Parameter | Mô tả | Ví dụ |
|-----------|-------------|---------|
| `name` | Ghi đè tên kết nối hiển thị trong sidebar | `?name=Production+DB` |

### SSL / TLS

| Parameter | Giá trị | Mô tả |
|-----------|--------|-------------|
| `sslmode` | `disable`, `prefer`, `require`, `verify-ca`, `verify-full` | Chế độ SSL (tên chuẩn PostgreSQL) |
| `tlsmode` | `0`–`4` | Chế độ SSL dạng số: 0=disable, 1=prefer, 2=require, 3=verify-ca, 4=verify-full |

**Ví dụ:**

```
postgresql://user:pass@host/db?sslmode=require
postgresql://user:pass@host/db?tlsmode=2
```

### Điều hướng

Các parameter này mở một bảng, view hoặc schema cụ thể ngay sau khi kết nối.

| Parameter | Mô tả | Ví dụ |
|-----------|-------------|---------|
| `table` | Mở bảng được đặt tên | `?table=users` |
| `view` | Mở view được đặt tên | `?view=active_orders` |
| `schema` | Chuyển sang schema này trước khi mở (chỉ PostgreSQL/Redshift) | `?schema=reporting` |

**Ví dụ:**

```
postgresql://user:pass@host/mydb?schema=reporting&table=monthly_sales
```

### Lọc dữ liệu

Tự động áp dụng bộ lọc cho bảng được mở.

| Parameter | Mô tả | Ví dụ |
|-----------|-------------|---------|
| `column` | Tên cột cần lọc | `?column=status` |
| `operation` | Toán tử lọc (ví dụ: `=`, `!=`, `LIKE`) | `?operation=LIKE` |
| `value` | Giá trị lọc | `?value=active` |
| `condition` | Điều kiện SQL WHERE thô (ghi đè column/operation/value) | `?condition=status%3D'active'+AND+role%3D'admin'` |

`raw` và `query` được chấp nhận là alias của `condition`.

**Ví dụ:**

```
postgresql://user:pass@host/mydb?table=orders&column=status&operation==&value=pending
postgresql://user:pass@host/mydb?table=orders&condition=total%3E1000
```

### Giao diện

| Parameter | Mô tả | Ví dụ |
|-----------|-------------|---------|
| `statusColor` | Gán màu cho kết nối (hex, khớp với màu gần nhất trong bảng màu) | `?statusColor=FF3B30` |
| `env` | Gán tag hiện có theo tên | `?env=production` |

**Ví dụ:**

```
postgresql://user:pass@host/db?statusColor=FF3B30&env=production
```

### MongoDB

| Parameter | Mô tả |
|-----------|-------------|
| `authSource` | Cơ sở dữ liệu xác thực (mặc định là `admin`) |

**Ví dụ:**

```
mongodb://user:pass@host:27017/mydb?authSource=admin
```

### Redis

Phần path đặt chỉ số database (0–15). Scheme `rediss://` tự động bật TLS.

```
redis://:password@host:6379/2       # database index 2
rediss://:password@host:6380        # TLS, index mặc định 0
```

### SSH Tunnel

| Parameter | Mô tả |
|-----------|-------------|
| `usePrivateKey` | `true` để sử dụng xác thực SSH bằng key thay vì mật khẩu |

**Ví dụ:**

```
postgresql+ssh://ubuntu@bastion/dbuser@10.0.0.5/mydb?usePrivateKey=true
```

## Cách Mở URL Trực tiếp Hoạt động

Khi bạn mở URL cơ sở dữ liệu từ trình duyệt hoặc terminal, TablePro xử lý tự động:

1. **Khớp kết nối đã lưu** — nếu host, port, database và username đều khớp với kết nối đã lưu, TablePro kết nối bằng kết nối đó (kể cả mật khẩu đã lưu)
2. **Không khớp** — TablePro tạo phiên tạm thời. Mật khẩu từ URL được lưu trong Keychain cho phiên đó nhưng kết nối không được lưu vào danh sách

Để mở URL từ terminal:

```bash
open "postgresql://user:pass@localhost:5432/mydb"
open "mysql://root@localhost/shop?table=products"
open "postgresql+ssh://ubuntu@bastion/user:pass@10.0.0.5/mydb?usePrivateKey=true"
```

<Note>
Để lưu kết nối vĩnh viễn, hãy dùng **Import from URL** trong biểu mẫu New Connection. Tính năng đó điền tất cả các trường để bạn xem xét rồi nhấn **Create**.
</Note>

## Ví dụ Đầy đủ

URL kết nối qua SSH, mở schema và bảng cụ thể, áp dụng bộ lọc trước, và đặt tag màu:

```
postgresql+ssh://deploy@bastion.example.com:22/dbuser:dbpass@10.0.1.5:5432/analytics?schema=public&table=events&column=type&value=purchase&statusColor=FF9500&env=staging
```

Phân tích chi tiết:

| Phần | Giá trị |
|------|-------|
| SSH user | `deploy` |
| SSH host | `bastion.example.com:22` |
| DB user / pass | `dbuser` / `dbpass` |
| DB host / port | `10.0.1.5:5432` |
| Database | `analytics` |
| Schema | `public` |
| Table | `events` |
| Bộ lọc | `type = purchase` |
| Màu | cam (`FF9500`) |
| Tag | `staging` |
