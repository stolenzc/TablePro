Kết quả query và nội dung bảng hiển thị trong grid kiểu bảng tính. Chỉnh sửa ô trực tiếp, sắp xếp cột, sao chép nhiều định dạng, và phân trang qua tập kết quả lớn.

## Xem Dữ Liệu

### Mở Bảng

Để xem nội dung bảng:

1. Mở rộng database trong sidebar
2. Click vào tên bảng
3. Data grid tải nội dung bảng

Hoặc chạy query:

```sql
SELECT * FROM users;
```

### Thông Tin Kết Quả

Header của data grid hiển thị:

- **Số hàng**: Số lượng hàng trả về
- **Thời gian thực thi**: Query mất bao lâu
- **Hàng bị ảnh hưởng**: Cho query UPDATE/DELETE

## Tính Năng Cột

### Thay Đổi Kích Thước Cột

- Kéo viền cột để thay đổi kích thước
- Nhấp đúp viền để tự động vừa chiều rộng
- Chiều rộng cột được ghi nhớ theo bảng

### Sắp Xếp Dữ Liệu

Click header cột để sắp xếp:

- **Click đầu tiên**: Tăng dần (A-Z, 0-9)
- **Click thứ hai**: Giảm dần (Z-A, 9-0)
- **Click thứ ba**: Bỏ sắp xếp

<Tip>
Sắp xếp chạy cục bộ trên dữ liệu đã tải. Để sắp xếp phía server, dùng `ORDER BY` trong query.
</Tip>

### Các Loại Cột

Mỗi kiểu dữ liệu được hiển thị phù hợp:

| Loại | Hiển thị |
|------|---------|
| Text/Varchar | Văn bản thuần |
| Integer | Số căn phải |
| Decimal/Float | Số được định dạng |
| Boolean | True/False |
| Date/Time | Chuỗi ngày định dạng |
| NULL | Chỉ báo "NULL" có style |
| JSON | JSON định dạng (mở rộng được) |
| ENUM | Popover dropdown |
| SET | Popover checkbox |
| Binary/BLOB | Chỉ báo nhị phân |

### Bộ Chọn Ngày/Giờ

Chỉnh sửa cột date, datetime, timestamp hoặc time mở popover chọn ngày gốc thay vì text field. Bộ chọn tự điều chỉnh theo kiểu cột:

| Loại Cột | Phần Tử |
|-------------|----------------|
| `DATE` | Năm, tháng, ngày |
| `DATETIME` / `TIMESTAMP` | Năm, tháng, ngày, giờ, phút, giây |
| `TIME` | Giờ, phút, giây |

Bộ chọn xử lý các định dạng ngày phổ biến và xuất giá trị theo chuẩn (`YYYY-MM-DD HH:MM:SS`). Dùng stepper hoặc nhập trực tiếp.

<Tip>
Bộ chọn ngày tự động phân tích giá trị hiện tại. Nếu không phân tích được, mặc định là ngày/giờ hiện tại.
</Tip>

### Tra Cứu Khóa Ngoại

Chỉnh sửa cột foreign key mở dropdown tìm kiếm hiển thị giá trị hợp lệ từ bảng tham chiếu:

- Dropdown hiển thị cả giá trị FK và cột mô tả (ví dụ: `1 — John Doe`)
- Gõ vào ô tìm kiếm để lọc
- Click đơn để highlight; nhấp đúp hoặc `Enter` để áp dụng
- Popover tự phát hiện cột hiển thị tốt nhất từ bảng tham chiếu

<Tip>
Tra cứu foreign key lấy tối đa 1.000 giá trị từ bảng tham chiếu. Với bảng lớn hơn, dùng ô tìm kiếm để thu hẹp.
</Tip>

### Trình Chỉnh Sửa Boolean

Nhấp đúp ô BOOLEAN, BIT, hoặc TINYINT(1) mở popover checkbox:

- **Checkbox**: Click để chuyển đổi true/false
- **OK/Cancel**: Click **OK** hoặc `Enter` để xác nhận; **Cancel** hoặc `Escape` để hủy
- **Cột nullable**: Trạng thái không xác định đại diện NULL

Hành vi theo database:

| Cơ sở dữ liệu | Loại cột | Giá trị |
|----------------|----------|---------|
| MySQL/MariaDB | `TINYINT(1)`, `BIT(1)` | `0` / `1` |
| PostgreSQL | `BOOLEAN` | `TRUE` / `FALSE` |
| SQLite | `INTEGER` | `0` / `1` |

<Tip>
Với cột boolean nullable, checkbox có ba trạng thái: checked (true), unchecked (false) và indeterminate (NULL).
</Tip>

### Trình Chỉnh Sửa Cột ENUM

Nhấp đúp ô ENUM mở dropdown tìm kiếm hiển thị tất cả giá trị được phép:

- **Ô tìm kiếm**: Gõ để lọc danh sách
- **NULL**: Hiển thị đầu danh sách cho cột nullable (chữ nghiêng)
- **Giá trị hiện tại**: Tô màu accent
- Click đơn để chọn và áp dụng ngay
- `Escape` để hủy

Hoạt động trên tất cả database hỗ trợ:

| Cơ sở dữ liệu | Nguồn ENUM |
|----------------|------------|
| MySQL/MariaDB | Kiểu `ENUM` gốc từ định nghĩa cột |
| PostgreSQL | Kiểu enum do người dùng định nghĩa (qua `pg_enum`) |
| SQLite | Ràng buộc `CHECK(column IN ('val1','val2'))` |

### Trình Chỉnh Sửa Cột SET

Nhấp đúp ô SET (MySQL/MariaDB) mở popover checkbox:

<Tip>
Nếu giá trị ENUM/SET không có sẵn (ví dụ từ kết quả query phức tạp thay vì duyệt bảng), ô chuyển về chỉnh sửa text inline.

### Trình Soạn Thảo JSON

Chỉnh sửa ô JSON hoặc JSONB mở popover editor JSON chuyên dụng:

- **Xác thực real-time**: Hiển thị JSON hợp lệ hay không khi bạn nhập
- **Save/Cancel**: Nút rõ ràng để lưu hoặc hủy

Editor xác thực JSON trước khi lưu. JSON không hợp lệ sẽ hiện hộp thoại xác nhận trước khi lưu.

<Tip>
Giá trị JSON được tự động compact (minified) khi lưu vào database.
</Tip>

### Lưu Thay Đổi

Thay đổi được xếp hàng, không áp dụng ngay. Quản lý thay đổi đang chờ:

- **Xem trước SQL**: Click **Preview SQL** (biểu tượng mắt) hoặc `Cmd+Shift+P` để xem SQL đang chờ
- **Commit**: Click **Commit** (hoặc `Cmd+S`) để áp dụng tất cả thay đổi
- **Discard**: Click **Discard** để hoàn tác tất cả
- **Undo/Redo**: `Cmd+Z` để undo, `Cmd+Shift+Z` để redo trước khi commit

Xem [Change Tracking](/features/change-tracking) để biết chi tiết.

<Warning>
Chỉnh sửa khả dụng cho query `SELECT * FROM table` đơn giản. Query với join hoặc aggregation là chỉ đọc.
</Warning>

### Thêm Hàng Mới

Để thêm hàng mới:

1. Click nút **+** trong toolbar (hoặc phím tắt)
2. Điền giá trị cho mỗi cột
3. Apply changes để lưu

### Xóa Hàng

Để xóa hàng:

1. Chọn hàng cần xóa
2. Click **Delete** hoặc phím `Delete`
3. Xác nhận
4. Apply changes để thực thi

## Chỉ Báo Thay Đổi

Thay đổi đang chờ có phản hồi trực quan:

- **Ô đã sửa** tô nền khác biệt
- **Hàng mới** hiển thị chỉ báo chèn
- **Hàng đã xóa** hiển thị chỉ báo xóa
- **Toolbar** hiển thị số thay đổi đang chờ

Xem lại tất cả sửa đổi trước khi commit. Xem [Change Tracking](/features/change-tracking) để biết chi tiết.

### Chế Độ Chi Tiết Hàng

Khi chọn một hàng, Cell Inspector hiển thị tất cả giá trị cột:

- **Danh sách cột có tìm kiếm**: Tìm nhanh cột bằng ô tìm kiếm
- **Tên cột và kiểu dữ liệu**: Mỗi mục hiển thị tên kèm kiểu dữ liệu
- **Giá trị đầy đủ**: Hiển thị không cắt ngắn
- **Chỉnh sửa nhiều dòng**: Cột TEXT/VARCHAR có vùng soạn thảo nhiều dòng
- **Chỉ báo chỉ đọc**: Ô không thể chỉnh sửa được đánh dấu rõ

Chỉnh sửa trực tiếp trong inspector. Thay đổi được xếp hàng giống inline editing, không áp dụng đến khi commit.

### Chế Độ Thông Tin Bảng

Khi không chọn hàng, Cell Inspector hiển thị metadata bảng:

| Metadata | Mô tả |
|----------|-------|
| Tên bảng | Tên bảng hiện tại |
| Số hàng | Tổng số hàng |
| Kích thước lưu trữ | Dung lượng đĩa |
| Ngày tạo | Thời điểm bảng được tạo |
| Engine | Storage engine (InnoDB, MyISAM, v.v.) |
| Charset | Bộ ký tự |
| Collation | Collation sắp xếp |

<Note>
Kết nối có Safe Mode đặt ở Read-Only và kết quả query phức tạp không cho phép chỉnh sửa. Chỉnh sửa bị tắt trong Cell Inspector cho những trường hợp đó. Xem [Safe Mode](/features/safe-mode) để biết chi tiết.
</Note>

## Điều Hướng

| Phím | Hành động |
|-----|--------|

| `Tab` | Ô tiếp theo |

| `Escape` | Hủy chỉnh sửa / Bỏ chọn |

- **Click**: Chọn ô
- **Nhấp đúp**: Chỉnh sửa ô

- **Scroll wheel**: Cuộn dọc
- **Shift + scroll**: Cuộn ngang

## Chọn

### Chọn Ô

- **Ô đơn**: Click vào ô
- **Phạm vi ô**: Click và kéo, hoặc click ô đầu rồi Shift+click ô cuối
- **Toàn hàng**: Click số hàng
- **Nhiều hàng**: Click số hàng và kéo, hoặc Cmd+click

### Sao Chép Dữ Liệu

1. Chọn ô hoặc hàng cần sao chép
2. Nhấn `Cmd+C`
3. Dán vào ứng dụng khác

Nhiều định dạng sao chép:

| Định dạng | Phím tắt | Mô tả |
|--------|----------|-------------|
| Tab-separated | `Cmd+C` | Mặc định, tương thích bảng tính |
| CSV | `Cmd+Shift+C` | Phân tách dấu phẩy |
| JSON | `Cmd+Option+C` | Mảng JSON |
| TSV | Menu ngữ cảnh | Tab-separated có header |

Tab-separated mặc định tương thích với:
- Excel
- Numbers
- Google Sheets
- Trình soạn thảo văn bản

## Phân Trang

## MongoDB Collections

Collection MongoDB có một số khác biệt so với SQL trong data grid.

### Schema Tài liệu

MongoDB không có schema cố định. TablePro suy luận cấu trúc cột bằng cách lấy mẫu tối đa 500 tài liệu. Nếu tài liệu có dạng khác nhau, một số cột hiển thị NULL cho tài liệu thiếu trường đó.

### Cột _id Không thể Chỉnh sửa

Cột `_id` không thể chỉnh sửa. MongoDB không cho phép sửa `_id` trên tài liệu hiện có. Muốn đổi `_id`, xóa rồi chèn lại.

### Hiển thị Kiểu BSON

Các kiểu BSON MongoDB được định dạng phù hợp:
- **ObjectId**: `ObjectId("...")`
- **Date**: ISO 8601 datetime
- **Binary**: `BinData(subtype, "base64...")`
- **Decimal128**: Chuỗi số thập phân
- **DBRef**: `DBRef("collection", id)`

### Xem trước MQL

Khi thay đổi (chèn, sửa, xóa) trong data grid, preview hiển thị lệnh MongoDB shell thay vì SQL:
- Chèn: `db.collection.insertMany([...])`
- Sửa: `db.collection.updateOne({filter}, {$set: {...}})`
- Xóa: `db.collection.deleteMany({_id: {$in: [...]}})`

## Làm Việc Với Dataset Lớn

### Mẹo Hiệu Suất

1. **Dùng LIMIT**: Luôn giới hạn kết quả khi khám phá
   ```sql
   SELECT * FROM large_table LIMIT 100;
   ```

2. **Chọn cột cụ thể**: Tránh `SELECT *` cho bảng nhiều cột
   ```sql
   SELECT id, name, email FROM users;
   ```

3. **Dùng phân trang**: Để data grid phân trang kết quả

4. **Lọc ở database**: Dùng WHERE thay vì tải tất cả
   ```sql
   SELECT * FROM orders WHERE status = 'pending';
   ```

### Sử Dụng Bộ Nhớ

Data grid tải theo trang để quản lý bộ nhớ:

- Chỉ trang đang hiển thị nằm trong bộ nhớ
- Cuộn tải trang mới khi cần
- Cột BLOB/TEXT lớn tải theo yêu cầu
